Introduction

This document defines the core schema for the Orchestration Architect (OA), focusing on the concepts of Object, Category, and Morphism, and their application in a mind map structure.

Core

oa:Object
An abstract entity foundational to the OA schema.
oa:Category
A structure consisting of objects and morphisms between them.
oa:Morphism
A directional relationship between objects in a category.

Example Graph Structure

@prefix oa: <urn:oa:ns#> .

oa:ExampleCategory a oa:Category ;
    rdfs:label "Example Category" ;
    oa:hasObject oa:Node1, oa:Node2 ;
    oa:hasMorphism oa:Edge1 .

oa:Node1 a oa:Object ;
    rdfs:label "Node 1" .

oa:Node2 a oa:Object ;
    rdfs:label "Node 2" .

oa:Edge1 a oa:Morphism ;
    rdfs:label "Link from Node 1 to Node 2" ;
    oa:source oa:Node1 ;
    oa:target oa:Node2 .
